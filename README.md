# 🛴 공유 킥보드 통합 안전 시스템

> Arduino 기반 공유 전동킥보드 안전 기능 연동 시스템  
> 남서울대학교 전자공학과 졸업 프로젝트

📄 **논문 게재** : JKIECS vol.21 no.1 (2026.02)  
🔗 **DOI** : [http://dx.doi.org/10.13067/JKIECS.2026.21.1.227](http://dx.doi.org/10.13067/JKIECS.2026.21.1.227)

## 프로젝트 소개

상용 킥보드(LGO-C250PRO)를 직접 분해·개조해서 헬멧 착용 감지, 방향지시등, 다중 탑승 방지, 헬멧 보관함 제어까지 4가지 안전 기능을 하나의 시스템으로 통합한 졸업 프로젝트입니다. 실제 주행 환경에서 직접 테스트하고 데이터를 측정했습니다.

## 만든 이유

질병관리청 2023년 통계에 따르면, 응급실에 실려온 킥보드 사고 환자의 **75%가 헬멧 미착용** 상태였습니다. 이는 헬멧 착용자 대비 약 **6.7배** 높은 수치입니다. 그러나 기존 공유 킥보드는 헬멧 착용을 강제하는 기술적 수단이 없었고, 다중 탑승이나 헬멧 분실 문제도 제도적으로만 다루어질 뿐 하드웨어 레벨에서의 해결책은 없었습니다.

이 문제를 센서와 마이크로컨트롤러를 활용해 기술적으로 해결하고자 이 시스템을 설계했습니다.

## 시스템 구성도

```
[ 헬멧 (Arduino Mega 2560) ]
  ├── TTP223B 터치 센서 x2   → 착용 감지
  ├── MPU-6050 자이로 센서   → 방향 감지
  ├── LCD 디스플레이         → 방향 표시
  └── HC-05 블루투스         → 킥보드 본체로 신호 전송
              │ BT
[ 킥보드 본체 (Arduino Uno) ]
  ├── BLC-160 모터 드라이버  → BLDC 모터 제어
  ├── SZH-SEN01 압력 센서 x4 → 다중 탑승 감지
  ├── KY-023 조이스틱        → 속도 입력
  └── HC-05 블루투스         → 앱 통신
              │ BT
[ 보관함 (Arduino Uno) ]
  ├── MFRC522 RFID 리더      → 헬멧 인식
  ├── A3144E 홀 센서         → 문 닫힘 감지
  └── ULN2003 스텝모터       → 잠금 제어
              │ BT
[ App Inventor 앱 ]
  └── 보관함 잠금/해제 버튼
```

## 주요 기능

### 1. 헬멧 착용 감지 + 모터 제어

<!-- 헬멧 센서 부착 사진 -->
![헬멧 착용 감지](screenshots/helmet_sensor.png)

헬멧 턱끈과 측두부에 정전식 터치 센서(TTP223B)를 각각 부착했습니다. 두 센서가 **동시에** 피부에 접촉될 때만 정상 착용으로 판단합니다. 한 곳만 닿는 경우(예: 헬멧을 손에 들고 있는 경우)는 미착용으로 처리해 오인식을 방지합니다.

착용 신호는 HC-05 블루투스를 통해 킥보드 본체의 Arduino Uno로 전달되고, Arduino Uno는 모터 드라이버(BLC-160)를 활성화해 주행을 허용합니다. 헬멧을 벗으면 급제동 대신 **전력 차단 방식**으로 관성 주행 상태로 전환합니다. 급정지 시 탑승자가 앞으로 쏠릴 수 있어 이 방식을 선택했습니다.

**센서 위치 최적화 과정**

처음에는 센서를 헬멧 측면(측두부)에만 부착했으나, 머리카락 간섭으로 인식률이 **24%** 에 그쳤습니다. 센서 위치를 턱끈으로 변경한 결과 인식률이 **92%** 로 향상됐습니다.

| 센서 위치 | 인식률 |
|-----------|--------|
| 측두부만 | 24% |            
| 턱끈 + 측두부 (최종) | 92% |


<img width="500" height="500" alt="터치센서위치에따른인식률" src="https://github.com/user-attachments/assets/2facc8b6-91ac-49b7-b5d8-77271c3a81c5" />



**E2E 응답시간 측정**

헬멧 탈착 → 모터 차단까지의 응답시간을 **240fps 고속카메라**로 촬영해 프레임 단위로 측정했습니다.

| 측정 항목 | 결과 |
|-----------|------|
| 평균 응답시간 | 397.9ms |
| 측정 방법 | 240fps 고속촬영 프레임 분석 |

### 2. 헬멧 방향지시등

<!-- 헬멧 LCD 방향 표시 사진 -->
![방향지시등](screenshots/turn_signal.png)

킥보드에는 사이드미러가 없어 뒤차가 방향 전환을 예측하기 어렵습니다. 특히 야간 주행 시 위험도가 높아집니다. 이를 해결하기 위해 헬멧 뒤쪽에 LCD를 부착하고, 조향 방향을 실시간으로 표시합니다.

자이로 센서(MPU-6050)의 **Z축 각속도를 적분**해 조향 방향을 계산하고, 방향에 따라 LCD에 `<<<<<` 또는 `>>>>>` 를 출력합니다.

딜레이를 0.24초로 설정했을 때 좌우 전환 표시가 가장 자연스럽고 직관적으로 출력됐습니다. 딜레이가 짧으면 미세한 흔들림에도 반응하고, 길면 실제 방향 전환보다 표시가 늦어졌습니다.

| 딜레이 설정 | 결과 |
|-------------|------|
| 0.10초 | 미세 흔들림에도 반응, 잦은 오작동 |
| **0.24초** | **가장 자연스러운 전환 (최종 채택)** |
| 0.40초 | 실제 전환보다 표시 지연 |

### 3. 다중 탑승 방지

<!-- 압력 센서 발판 배치 사진 -->
![다중 탑승 방지](screenshots/dual_ride.png)

킥보드 발판 네 귀퉁이에 압력 센서(SZH-SEN01)를 배치했습니다. 1인이 탑승할 경우 4개 센서를 동시에 모두 밟기 어렵도록 센서 간격을 설계했습니다. 4개 센서가 모두 임계값(약 50)을 초과하면 2인 이상 탑승으로 판단하고 부저를 울립니다.

모터를 강제 정지하는 방안도 검토했으나, 주행 중 갑작스러운 정지는 2명 모두에게 더 위험할 수 있어 **경고음 방식**을 채택했습니다.

| 항목 | 내용 |
|------|------|
| 감지 방식 | 4개 압력 센서 동시 임계값 초과 |
| 임계값 | 약 50 |
| 대응 방식 | 부저 경고음 출력 |
| 감지 정확도 | 약 95% |

### 4. 헬멧 보관함 제어

<!-- 헬멧 보관함 외관 및 내부 사진 -->
![헬멧 보관함](screenshots/helmet_box.png)

헬멧에 RFID 태그를 부착하고, 보관함 내부에 MFRC522 리더기와 A3144E 홀 센서를 설치했습니다. 헬멧을 보관함에 넣고 문을 닫으면 두 조건이 모두 충족될 때만 App Inventor로 제작한 앱에서 잠금 버튼이 활성화됩니다.

**이중 확인 조건**
- RFID 인식: 헬멧 태그가 리더기 3.6cm 이내에 있을 것
- 홀 센서 감지: 자석이 0.5cm 이내로 근접할 것 (문이 완전히 닫혔을 때)

두 조건을 모두 충족해야만 잠금이 활성화되므로, 헬멧 없이 문만 닫거나 RFID 태그만 가져다 댄 경우는 잠금 버튼이 뜨지 않습니다.

<!-- 앱 화면 스크린샷 -->
![보관함 제어 앱](screenshots/app_lock.png)

| 센서 | 유효 감지 거리 |
|------|----------------|
| MFRC522 RFID | 3.6cm 이내 |
| A3144E 홀 센서 | 0.5cm 이내 |

## 블루투스 통신 범위 테스트

헬멧과 킥보드 본체 간 HC-05 블루투스 통신 거리별 성공률을 측정했습니다.

| 거리 | 통신 성공률 |
|------|-------------|
| 5m | 100% |
| 10m | 100% |
| 15m | 100% |
| 20m | 50% |
| 25m | 25% |

**15m 이내에서 안정적**으로 동작합니다. 킥보드와 헬멧 간 거리가 20m 이상 벌어지면 통신이 끊겨 모터도 자동으로 차단됩니다. 헬멧 없이 킥보드만 가져가는 상황을 어느 정도 방지할 수 있습니다.

| A3144E 홀 센서 | 보관함 문 닫힘 감지 |
| BLC-160 모터 드라이버 | BLDC 모터 제어 |
| KY-023 조이스틱 | 속도 입력 |
| ULN2003 스텝모터 | 보관함 잠금 장치 구동 |

**전원 분리 설계**

모터·드라이버 계통은 **24V**, Arduino·블루투스 계통은 **9V**로 전원을 분리했습니다. 초기 단일 전원 구성 시 노이즈 간섭으로 인해 센서 오작동이 발생해 계통 분리를 적용했습니다.

## 한계 및 개선 방향

**통신 범위**  
HC-05 블루투스는 15m 이상에서 불안정합니다. BLE 또는 Wi-Fi 기반 모듈로 교체하면 통신 범위를 크게 늘릴 수 있을 것으로 봅니다.

**압력 센서 임계값**  
현재는 고정 임계값(약 50)을 사용하기 때문에 탑승자 체중 차이에 따라 오인식이 발생할 수 있습니다. 탑승자별 기준값을 학습하거나 동적으로 조정하는 방식을 도입하면 정확도를 높일 수 있습니다.

**서버 연동 부재**  
현재는 단일 킥보드 기준의 로컬 구현입니다. 실제 공유 킥보드 서비스에 적용하려면 중앙 서버와의 연동 구조(헬멧 착용 이력, 다중 탑승 경고 로그 수집 등)가 필요합니다.
