# 공유 킥보드 통합 안전 시스템 (Arduino)

졸업 프로젝트로 진행한 공유 전동킥보드 안전 기능 연동 시스템입니다.  
상용 킥보드(LGO-C250PRO)를 직접 분해·개조해서 실제 주행 환경에서 테스트했습니다.

논문은 JKIECS vol.21 no.1 (2026.02)에 게재되었습니다.  
DOI: http://dx.doi.org/10.13067/JKIECS.2026.21.1.227

---

## 만든 이유

질병관리청 2023년 통계에 따르면, 응급실에 실려온 킥보드 사고 환자의 75%가 헬멧을 안 쓴 상태였습니다. 착용자보다 약 6.7배 높은 수치입니다. 그런데도 기존 공유 킥보드는 헬멧 착용을 강제하는 수단이 없었고, 다중 탑승이나 헬멧 분실 문제도 그냥 방치되는 경우가 많았습니다.

이걸 기술적으로 해결해보고 싶었습니다.

---

## 기능 설명

### 1. 헬멧 착용 감지 + 모터 제어

헬멧 턱끈과 측두부에 정전식 터치 센서(TTP223B)를 붙여서, 두 센서가 동시에 피부에 닿았을 때만 "정상 착용"으로 판단합니다. 이 신호를 블루투스로 킥보드 본체에 전달하면, Arduino Uno가 모터 드라이버를 활성화합니다.

헬멧을 벗으면 급제동이 아니라 전력을 끊어서 관성 주행 상태로 전환합니다. 급정지하면 오히려 넘어질 수 있어서 이 방식을 선택했습니다.

처음엔 센서를 헬멧 옆면에 달았다가 머리카락 간섭으로 인식률이 24%밖에 안 나왔습니다. 위치를 턱끈으로 바꾸니 92%까지 올라갔습니다.

E2E 응답시간은 240fps 고속촬영으로 측정했고 평균 397.9ms였습니다.

### 2. 헬멧 방향지시등

자이로 센서(MPU-6050)로 Z축 각속도를 적분해서 조향 방향을 계산하고, 헬멧 뒤에 달린 LCD에 `<<<<<` 또는 `>>>>>` 로 방향을 표시합니다.

킥보드는 사이드미러가 없어서 뒤차가 방향 전환을 예측하기 어렵습니다. 특히 야간에는 더 위험하고요.

딜레이를 0.24초로 설정했을 때 좌우 전환이 가장 자연스럽게 출력됐습니다.

### 3. 다중 탑승 방지

발판 네 귀퉁이에 압력 센서(SZH-SEN01)를 설치했습니다. 센서 하나하나는 한 명이 서있을 때 동시에 다 밟기 어렵도록 배치했고, 4개 모두 임계값(약 50)을 넘으면 2인 이상 탑승으로 판단해서 부저를 울립니다.

다중 탑승 감지 시 모터를 강제로 멈추는 방식도 검토했지만, 두 명 다 위험해질 수 있어서 경고음만 내는 방식을 택했습니다.

정확도는 약 95%였습니다.

### 4. 헬멧 보관함 제어

헬멧에 RFID 태그를 붙이고, 보관함 안에 MFRC522 리더기와 홀 센서를 설치했습니다. 헬멧을 넣고 문을 닫으면 RFID 인식 + 홀 센서 감지 두 조건이 동시에 충족될 때만 App Inventor로 만든 앱에서 잠금 버튼이 활성화됩니다.

RFID는 3.6cm 이내, 홀 센서는 0.5cm 이내에서 안정적으로 동작했습니다. 이 거리를 기준으로 보관함 내부 구조를 설계했습니다.

## 하드웨어

```
Arduino Mega 2560     - 헬멧 측 메인 컨트롤러
Arduino Uno (x2)      - 킥보드 본체 / 보관함 제어
HC-05 블루투스 (x3)   - 헬멧 ↔ 킥보드 ↔ 앱 통신
TTP223B 터치 센서     - 헬멧 착용 감지
MPU-6050 자이로       - 방향 감지
SZH-SEN01 압력 센서   - 다중 탑승 감지
MFRC522 RFID 리더     - 헬멧 인식
A3144E 홀 센서        - 보관함 문 닫힘 감지
BLC-160 모터 드라이버 - BLDC 모터 제어
KY-023 조이스틱       - 속도 입력
ULN2003 스텝모터      - 보관함 잠금
```

전원은 모터·드라이버 계통은 24V, Arduino·블루투스 계통은 9V로 분리했습니다. 노이즈 간섭 때문에 한 전원으로 묶으면 센서 오작동이 생겼습니다.

## 블루투스 통신 범위

| 거리 | 성공률 |
|------|--------|
| 5m | 100% |
| 10m | 100% |
| 15m | 100% |
| 20m | 50% |
| 25m | 25% |

15m 이내에서는 안정적입니다. 킥보드와 헬멧이 20m 이상 벌어지면 통신이 끊겨서 모터도 차단됩니다. 헬멧 없이 킥보드만 가져가는 상황을 어느 정도 막을 수 있습니다.

---

## 한계 및 개선 방향

- 블루투스(HC-05)는 15m 이상에서 불안정합니다. BLE나 Wi-Fi 기반으로 교체하면 범위를 늘릴 수 있을 것 같습니다.
- 압력 센서 임계값이 고정값이라서 탑승자 체중 차이에 따라 오인식이 생길 수 있습니다.
- 현재는 단일 킥보드 기준 구현이라서 실제 공유 서비스에 붙이려면 서버 연동 구조가 필요합니다.
